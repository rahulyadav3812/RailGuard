1. ABSTRACT
──────────────────────────────────────────────────────────
This project presents a fog computing-based security model
for detecting data manipulation attacks in railway signaling
and control networks. The system implements a 3-tier
architecture (Edge-Fog-Cloud) with ensemble machine learning
detection deployed at the fog layer for low-latency threat
identification. Seven detection models are trained and
evaluated against six railway-specific attack types.

2. SYSTEM ARCHITECTURE
──────────────────────────────────────────────────────────
Tier 1 - Edge Layer (5 devices):
  - Signal Controller (S1)
  - Track Circuit Monitor (TC1)
  - Points Machine (P1)
  - Axle Counter (AC1)
  - Eurobalise (B1)

Tier 2 - Fog Layer (3 nodes):
  - Station Fog Node (real-time ML detection)
  - Junction Fog Node (real-time ML detection)
  - Lineside Fog Node (real-time ML detection)

Tier 3 - Cloud Layer (1 server):
  - Centralized monitoring
  - Model retraining
  - Alert aggregation

Security: AES-256 encryption + RSA-2048 key exchange
Compliance: ERTMS/ETCS, EN 50129, IEC 62443

3. DATASET
──────────────────────────────────────────────────────────
Total Samples:    15,000
  Normal:         10,000 (66.7%)
  Attack:          5,000 (33.3%)

Train/Test Split: 80% / 20%
  Training Set:   12,000 samples
  Testing Set:     3,000 samples

Features Engineered: 20
  - Signal state, speed, track occupancy
  - Deviation metrics (norm, percentage)
  - Network metrics (latency, packet size)
  - Integrity checks (hash validation)
  - Safety consistency (signal-speed, track-signal)

5. DETECTION MODELS
──────────────────────────────────────────────────────────
  1. Statistical Anomaly Detector
     - Z-score based, threshold = 2.5
     - Baseline from normal training data

  2. Support Vector Machine (SVM)
     - RBF kernel, auto kernel scale
     - Posterior probability calibration

  3. Random Forest
     - 100 trees, min leaf size = 5
     - OOB prediction & importance ranking

  4. K-Nearest Neighbors (KNN)
     - K=5, squared-inverse distance weighting
     - Standardized features

  5. LSTM Neural Network
     - 2-layer LSTM (128 + 64 units)
     - Dropout 0.3, Adam optimizer
     - 30 epochs training

  6. Rule-Based IDS
     - 20 railway-specific safety rules
     - Weighted violation scoring

  7. Weighted Ensemble
     - Weights: [0.15, 0.15, 0.35, 0.10, 0.15, 0.10]
     - Decision threshold: 0.35
     - RF-weighted for highest accuracy

The fog computing security model successfully demonstrates:

  2. COMPREHENSIVE COVERAGE: All six attack types are
     detected with high confidence, with Random Forest

     threat detection within safety-critical timeframes.

  4. SCALABLE ARCHITECTURE: The 3-tier design supports
     distributed deployment across railway networks with
     local ML inference at each fog node.

  5. MULTI-MODEL APPROACH: Combining statistical, ML,
     deep learning, and rule-based methods provides
     defense-in-depth against diverse attack vectors.
